## Contract

Output frame time is `outputIndex × 1000 / 30`; timestamps are never generated by repeated floating-point addition. Frame count is `floor(sourceDurationMs × 30 / 1000) + 1`, so frame zero exists and the last output timestamp never exceeds the last CDP timestamp.

For each output timestamp, the single-pass selector retains the latest source at or before that time and inspects the first source after it. The smaller timestamp distance wins, and an exact tie selects the earlier frame. Selection ignores receive time, filesystem metadata, source array position, average cadence, and cursor timing. No pixels are interpolated.

The capture reader uses Node streams and line reading. It validates source order, strict timestamps, consecutive indices, canonical filenames, manifest counts and dimensions, viewport and scale invariants, and file existence. It calls filesystem metadata operations for JPEG existence but never opens or reads JPEG content. The plan contains source references only.

Full-plan median and p95 diagnostics use deterministic bounded P² estimators. Exact distributions are retained only for the 30 event mappings. This keeps resampling memory independent of capture length.

## Risks for Day 5

- Browser pixels can lead or lag output time by up to 11.577 ms in this capture. Cursor and camera metadata must still be evaluated at output time, not source time.
- The largest source-to-mouse-down difference was 19.337 ms. Day 5 must preserve it rather than shifting click semantics.
- This high-cadence capture required no repeated source frame. Reuse behavior is unit-tested, but the compositor must handle repeated references for lower-cadence captures.
- P² full-plan quantiles are bounded estimates; minima and maxima and all 30 click diagnostics are exact.
- Source file paths in plan records are relative to the source capture bundle. The compositor must resolve them against `sourceCapturePath` without copying image data.
